
To program a Tiva C series microcontroller through a Bash terminal in a Linux environment, you will need 
an ARM cross-compiler toolchain, the TivaWare library, and a command-line flash programming utility like lm4tools
(or sflash) or OpenOCD.

The general process involves three main stages: setting up the environment, building your code using a Makefile, 
and flashing the compiled binary to the board. 
Prerequisites 

• A Linux environment with Bash (e.g., Ubuntu, or  Windows Subsystem for Linux (WSL)). 
• An ARM GCC cross-compiler: Install with  on Ubuntu. 
• lm4tools (for flashing via the ICDI interface) or OpenOCD (for more advanced debugging/flashing). 
• TivaWare C Series software library 
 downloaded and unzipped locally. 

 Step 1: Set up the Development Environment 

 	1. Install the ARM Toolchain: 
 	sudo apt install gcc-arm-none-eabi binutils-arm-none-eabi libnewlib-arm-none-eabi

	2. Install Flashing Tools: 

 	• For lm4tools (a common choice for the LaunchPad's ICDI): You may need to compile it from source or use a 
	package manager if available for your distribution. 
		• Alternatively, install OpenOCD: sudo apt install openocd. 

 	3. Organize TivaWare: Download the TivaWare exe  file (it's a self-extracting archive that works in Linux) 
		from the  TI website, move it to your projects directory, and unzip it. 

		cd ~/tiva-projects
		mkdir tivaware
		mv ~/Downloads/SW-TM4C-*.exe tivaware
		cd tivaware
		unzip SW-TM4C-*.exe


 Step 2: Compile Your Project via Bash and Make [2]  
		You will need a  to manage the compilation process, which is a standard method for building projects in
		a Linux environment. 

		1. Create a Project Structure: 
			cd ~/tiva-projects
			mkdir my_blinky_project
			cd my_blinky_project
			# Create your main.c file and a Makefile here
		
		2. Write the Makefile: The Makefile needs to define the correct compiler flags (e.g., -mcpu=cortex-m4 -mthumb) 
		and link the necessary TivaWare driver library (driverlib.a).
			# Adjust paths based on your TivaWare installation location
			TIVAWARE_PATH = ~/tiva-projects/tivaware/TivaWare_C_Series-x.y.z
			COMPILER = arm-none-eabi-gcc
			# ... other flags and rules ...


		Example snippet for a Makefile (paths need adjustment): 
		(You can find complete Makefile examples on GitHub repositories dedicated to Tiva C projects in Linux). 
		3. Build the Project: Run the make  command in your terminal. This generates an executable binary file 
		(often with a .axf or .bin extension). 
			make all

Step 3: Flash the Binary from the Bash Terminal 
		Once you have your compiled binary, use a command-line utility to flash it to the board. [15]  
		Using lm4tools
		lm4flash  (part of the lm4tools suite) is designed specifically for the Tiva LaunchPad's 
		integrated debugger.

		# Example command to flash your program.axf file
		lm4flash program.axf

		Using sflash (via UART bootloader) 
		If using the UART bootloader functionality built into the Tiva C series, you can use the sflash utility 
		found within the TivaWare installation's tools folder. This is ideal for remote flashing or automated 
		scripting.

		# Example command using sflash, adjust path and port as necessary
		~/tiva-projects/tivaware/tools/bin/sflash -u /dev/ttyUSB0 program.bin


		Using OpenOCD
		OpenOCD provides a robust interface for flashing and debugging. This is often used in combination with 
		GDB for full command-line debugging sessions.

		# Start OpenOCD in the background with the appropriate configuration file for your board
		openocd -f board/ti_ek-tm4c123gxl.cfg &

		# Use arm-none-eabi-gdb to connect to OpenOCD and load the program
		arm-none-eabi-gdb program.axf -ex "target remote localhost:3333" -ex "load" -ex "quit"

		By scripting these make and flashing commands in a Bash script, you can automate your entire build-and-flash cycle.
